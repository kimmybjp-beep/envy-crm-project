Back Office Admin Manual

คู่มือ Admin ENVY Reward CRM

สำหรับทีม T&G Back Office ใช้ดูการอนุมัติร้าน, reset password, QR Generator, reward fulfillment, dashboard และ export report

Login Store QR Generator Rewards Dashboard Export LINE Bot

1. การเข้าใช้งาน Admin

  1. เข้า https://envy-crm-project.vercel.app/admin/login
  2. กรอก Admin Password
  3. ระบบจะพาไปหน้า /admin สำหรับจัดการร้านค้า

หน้า Admin ไม่ควรส่งให้ร้านค้า ใช้เฉพาะทีม Back Office เท่านั้น

2. Store Management

2.1 อนุมัติหรือปฏิเสธร้านค้า

  1. เข้าเมนู Stores หรือ /admin
  2. ดูจำนวนร้านที่รออนุมัติใน Pending queue
  3. ตรวจชื่อร้าน, เจ้าของร้าน, เบอร์โทร และรูปหน้าร้าน
  4. กด Approve ถ้าข้อมูลถูกต้อง
  5. กด Reject ถ้าข้อมูลไม่ผ่าน

2.2 Reset Password ให้ร้านค้า

  1. หาแถวของร้านที่ต้องการ reset
  2. พิมพ์ Password ใหม่ในช่อง New password
  3. กด Reset
  4. แจ้งร้านให้ Login ด้วยเบอร์โทรเดิมและ Password ใหม่

2.3 Store Login Popup Message

  1. พิมพ์ข้อความประกาศในช่อง Store Login Popup Message
  2. กด Publish Popup
  3. ข้อความจะเด้งหลังร้านค้า Login และแสดงบนหน้า Home

2.4 Suspicious Scan Alerts

ถ้าระบบพบพฤติกรรมผิดปกติ เช่น QR ซ้ำหรือเข้ากฎ fraud ระบบจะสร้าง alert ให้ Admin ตรวจสอบ

  • อ่านรายละเอียด alert
  • ตรวจชื่อร้านและ QR code ที่เกี่ยวข้อง
  • แจ้งทีมภาคสนามตรวจสอบถ้าจำเป็น
  • เมื่อเคลียร์แล้วกด Resolve

3. QR Generator

  1. เข้าเมนู QR หรือ /admin/qr-generator
  2. กรอก Distributor name
  3. กรอก Distributor code ถ้าต้องการกำหนดเอง
  4. กรอกจำนวน QR ที่ต้องการ
  5. เลือก Apple size เช่น 24, 30, 36
  6. ตั้ง Point value, Campaign name และ Sticker color
  7. กด Generate ZIP
ข้อมูลเก็บที่ไหนใช้ทำอะไร
QR Batchqr_batchesเก็บรอบการสร้าง QR, distributor, size, campaign, จำนวน
QR Code รายตัวqr_codesเก็บ QR ID, URL, point value, status generated/claimed
เลขรันต่อเนื่องqr_code_countersกันเลขซ้ำใน distributor/size เดียวกัน

หลัง Generate สำเร็จ Browser จะดาวน์โหลด ZIP ที่มีไฟล์ SVG sticker และ CSV summary ไว้ในเครื่อง

4. Reward Management

4.1 เพิ่มของรางวัล

  1. เข้าเมนู Rewards หรือ /admin/rewards
  2. กรอก Reward name
  3. กรอก Description
  4. ใส่ Points required และ Stock
  5. เลือก Active reward ถ้าต้องการให้ร้านเห็น
  6. กด Save Reward

4.2 จัดการคำขอแลกรางวัล

  • PENDING ร้านส่งคำขอแลกเข้ามาแล้ว
  • PROCESSING กำลังตรวจสอบหรือเตรียมส่งของ
  • PAID ทำจ่ายหรือดำเนินการแล้ว
  • SHIPPED ส่งของแล้ว
  • CANCELLED ยกเลิกคำขอ

5. Interactive Dashboard

Dashboard ใช้ดูภาพรวมการทำงานของระบบแบบ live จาก database

Total Storesจำนวนร้านทั้งหมดตาม filter
Approved / Pendingจำนวนร้านที่อนุมัติแล้วและรออนุมัติ
QR Issuedจำนวน QR ที่สร้างในระบบ
QR Claimedจำนวน QR ที่ร้านสแกนสะสมแต้มแล้ว
Claim Rateเปอร์เซ็นต์ QR ที่ถูก claim เทียบกับที่สร้าง
Pending Fulfillmentจำนวนคำขอแลกรางวัลที่ยังค้าง

Filter ที่ใช้ได้

  • By day
  • By week
  • By month
  • All time
  • By Tier

กราฟ QR Market Landing Graph แสดง flow: Distributor → Apple Size/Campaign → Store ที่ claim QR แล้ว

6. Data Export

เข้าเมนู Data หรือ /admin/database เพื่อดาวน์โหลด CSV เปิดใน Excel

Reportใช้ดูอะไร
QR Claim ReportQR ที่ถูก claim, ร้านที่ claim, แต้ม, วันที่ claim, distributor และ size
QR Inventory ReportQR ทั้งหมดที่สร้าง, batch, campaign, status และ URL
Reward Fulfillment Reportรายการแลกรางวัล, ร้านค้า, reward, แต้มที่ใช้ และสถานะจัดส่ง/ทำจ่าย
Raw Table AuditExport ตารางดิบ เช่น stores, scans, qr_codes, rewards สำหรับตรวจเชิงเทคนิค

7. LINE Bot และ Daily Summary

  • หน้า Dashboard มีปุ่ม Send LINE Summary สำหรับส่งสรุปเข้า LINE ทันที
  • ระบบรองรับ Vercel Cron สำหรับส่งสรุปรายวัน
  • บอทใน group จะเงียบถ้าไม่ได้พิมพ์คำสั่งที่กำหนด

คำสั่งหลัก

  • summary หรือ สรุป เพื่อขอสรุป dashboard
  • groupid หรือ ขอรหัสกลุ่มหน่อย เพื่อดู group id
